카카오뱅크를 선택한 이유는 기술이 서비스의 본질인 조직이기 때문입니다. 기존 은행과 달리 수천만 고객의 금융 데이터를 실시간으로 정확하게 처리하는 기술 인프라 위에 모든 서비스가 세워지며, 엔지니어의 결정이 직접 고객 경험으로 이어집니다. 특히 금융회계 시스템은 단순한 데이터 처리를 넘어, 분산 이벤트 환경에서의 트랜잭션 정합성과 대규모 데이터의 일관성이 핵심인 도메인입니다. 이는 제가 실전에서 가장 깊이 다룬 문제와 직접 맞닿아, 지금 당장 기여할 수 있는 역량을 가진 곳이라 확신했습니다.

## Q2. 같지만 다른 은행인 카카오뱅크, 본인의 차별화 된 역량 및 기술을 근거로 지원한 직무에 적합한 이유를 기술해 주세요.

> [글자수: 약 820 / 1000]

지원 직무에 제가 적합한 이유를 세 가지 역량을 근거로 말씀드리겠습니다.

**첫째, 실전 금융 데이터 정합성 경험입니다.** QR 결제 중계 시스템에서 분산 환경의 이벤트 유실·중복 처리·상태 불일치 문제를 직접 해결했습니다. Transactional Outbox 패턴 도입으로 파일 유실률 100%→0%, Final API 패턴으로 거래 상태 불일치 95% 감소를 달성했습니다. 금융회계 시스템의 핵심인 데이터 정합성 문제를 상용 서비스에서 직접 해결한 경험입니다.

**둘째, 지표 기반 성능 최적화 역량입니다.** Redis Cache-Aside 패턴과 트랜잭션 경계 최적화로 API 응답시간 85% 개선(200ms→30ms), DB I/O를 Polling 1회당 최대 120회→0회로 줄였습니다. Kafka Request-Reply 아키텍처에서는 인스턴스별 고유 Reply 토픽과 CorrelationId 기반 라우팅으로 분산 환경 타임아웃 문제를 완전히 해소했습니다. 감이 아닌 Pinpoint APM 측정 기반의 수치로 증명되는 개선이 제 일하는 방식입니다.

**셋째, 금융 컴플라이언스 실무 경험입니다.** PCI DSS 자격 취득 프로젝트에서 HashiCorp Vault 기반 시크릿 관리 체계, Keycloak 2FA, JPA Auditing+Kafka 기반 감사 로그 추적 체계를 직접 구축했습니다. 금융당국 선불 라이센스 취득에도 핵심 역할을 맡아, 높은 보안·감사 기준이 요구되는 금융 시스템 환경에 이미 익숙합니다.

세 가지 역량 모두 화이트보드 위의 이론이 아닌, 상용 서비스에서 직접 검증한 경험입니다. 카카오뱅크 금융회계 시스템에서 즉각적인 기여를 할 수 있다고 자신합니다.
