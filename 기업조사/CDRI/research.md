# CDRI (씨디알아이) 기업 조사 리포트
작성일: 2026-05-12
지원 직무: 백엔드 개발자 (서비스개발본부 플랫폼1팀)

---

## 1. 회사 개요

- **설립:** 2020년 (초기 사명: 주식회사 30코스 → 2022년 씨디알아이로 변경)
- **대표:** 김광일 (화장품 브랜드사 SCM 총괄 출신)
- **본사:** 서울 강남구
- **규모:** 약 58명 (2026년 기준, 원티드 공개 수치)
- **상장 여부:** 비상장
- **한 줄 요약:** 화장품 산업(TIC: Testing, Inspection, Certification)의 비효율을 IT·AI로 자동화하는 뷰티 테크 B2B 플랫폼

---

## 2. 비즈니스 이력 (과거 → 현재)

### 창업기 (2020 ~ 2021)
- 김광일 대표가 화장품 브랜드사(팜스킨) SCM 총괄 근무 중 인허가 프로세스의 비효율을 직접 경험
  - 수출국마다 서류를 반복·중복 제출해야 하는 구조 → "이걸 자동화할 수 없을까"에서 창업
- 초기 사명 '30코스'로 설립, 화장품 인허가 자동화 서비스 개발 시작

### 성장기 (2022 ~ 2023)
- 사명을 **씨디알아이(CDRI)**로 변경, 브랜드 정비
- **CERTICOS** 정식 론칭: 36개국 화장품 해외 수출 인허가 자동화 플랫폼
  - 유럽 CPNP 인증 기간 **10일로 단축** (업계 평균 대비 대폭 단축)
  - 론칭 6개월 만에 누적 인허가 **2,000건 돌파**
- 한번의 서류로 35개국 인증 완료하는 구조 구축 → K-뷰티 데이터화 선도
- **해외규격인증획득지원사업** 수행기관 선정 (정부 사업)
- AI 스크리닝·딥러닝 기반 화장품 인증관리 시스템 **특허 출원**
- 매출: 약 19억 (2023년)

### 현재 (2024 ~ )
- 누적 고객사 **1,400개+**, 임상시험 컨설팅 **3,700건+**, 특허 등록 **10건**
- 매년 평균 **2.5배 이상** 매출 성장 지속
- **K-Global@실리콘밸리 2024** 참가 → AI·빅데이터 기반 글로벌 진출 가속
- **CertiPT** 출시: LLM + 멀티에이전트 기술로 인허가 원천 서류 자동 분류·분석
- **Cosmecarrer** 신사업: 화장품 업종 특화 채용 솔루션 (2025~ 런칭 준비)
- 2024년 **글로벌 VC 투자 유치** (시리즈·금액 미공개)
- 외부 어드바이저로 SK텔레콤·삼성전자·뷰노 출신 기술진 영입

## 4. 현재 기술 환경 & 도전 과제

### 확인된 기술 스택 (JD + 테크블로그 기준)
- **백엔드:** Java, Spring Boot, JPA, MySQL / MariaDB
- **인프라:** AWS (클라우드 기반 운영), Docker
- **데이터:** Redis (우대), MongoDB (우대)
- **AI:** AI 스크리닝, LLM + 멀티에이전트 (CertiPT)
- **협업:** Jira (이슈 트래킹), Slack (커뮤니케이션 분리), git flow 전략
- **프로세스:** 애자일 스크럼 (데일리 스크럼), 코드 리뷰, 테크톡

### 현재 풀고 있는 기술 과제
1. **도메인 복잡도 관리:** 화장품 인허가는 국가마다 규정이 달라 데이터 스키마와 비즈니스 규칙이 복잡 → 변화에 빠르게 대응할 수 있는 데이터 설계 필요
2. **AI 기능 연동:** AI 부서와 협업하여 LLM/AI 결과를 백엔드 API로 통합 — AI-Backend 인터페이스 설계가 핵심
3. **전산화되지 않은 데이터 자동화:** 종이·PDF 기반 인허가 서류를 구조화된 데이터로 전환하는 파이프라인 구축 중
4. **권한 기반 서비스 복잡성:** 고객사·서비스별 복잡한 권한 체계 구현 및 개선
5. **인프라 안정화:** Certicos + Cosmetest 동시 운영, AWS 기반 자동화 배포

## 5. 내가 기여할 수 있는 영역

| 회사의 과제 / 필요 | 내 경험 / 역량 | 구체적 기여 방식 |
|-------------------|----------------|-----------------|
| AI 기능 백엔드 연동 (CertiPT API 설계) | AWS Bedrock + Claude LLM 자동화 배치 설계·구현 → 수동 업무 월 100건→5건(95%↓) | AI 부서 협업 구조 이미 경험, LLM 응답 처리 API 설계에 즉시 투입 가능 |
| Kafka 기반 비동기 파이프라인 필요 | Kafka Request-Reply, MSA Push 아키텍처 설계 (타임아웃 0%, 중복 Push 0건) | 인허가 서류 자동화 파이프라인, AI 처리 결과 비동기 전달 구조에 적용 가능 |
| Redis 활용 (우대 사항) | Redis Cache-Aside + 트랜잭션 경계 최적화 → API 응답 85%↓ | DB 부하 감소, 고객사 증가에 따른 캐시 전략 설계 기여 |
| 데이터 정합성 보장 (인허가 서비스) | Outbox 패턴, Final API 패턴으로 파일 유실 0건·CS 95%↓ 달성 | 인허가 원천 서류 처리 실패 시 유실 방지, 재처리 구조 설계 |
| AWS 기반 인프라 운영 (우대 사항) | AWS 실무 운영, Docker, Jenkins CI/CD 파이프라인 구축 경험 | 배포 자동화, 장애 감지 구조 개선에 기여 |
| 초기 설계→운영 전 주기 (우대 사항) | QR 결제 중계 시스템 0→1 구축·운영 경험 (거래 100배 성장 주도) | 빠르게 성장 중인 CDRI에서 신규 서비스(Cosmecarrer 등) 설계부터 담당 가능 |

## 6. 면접 활용 포인트

### 기술면접에서 쓸 것

- **도메인 복잡도 경험:** "결제 도메인도 국가·파트너사별 규칙이 복잡하다. 화장품 인허가처럼 국가마다 규정이 다른 데이터를 다루는 경험이 있다."
- **AI-Backend 연동:** "AWS Bedrock 기반 LLM 자동화 배치를 설계·구현한 경험이 있다. AI 부서와 협업하여 AI 결과를 백엔드 API로 통합하는 구조에 바로 기여할 수 있다."
- **Redis 우대 사항 직접 답변 가능:** Cache-Aside + 트랜잭션 경계 설계로 DB I/O 120건→0건, 응답 85% 개선 수치로 답변 가능
- **데이터 정합성:** Outbox 패턴, Final API 패턴으로 분산 환경에서 데이터 유실 0건 달성 경험 — 인허가 서비스의 정합성 요구사항과 직접 연결

### 인성·컬처핏 면접에서 쓸 것

- **왜 CDRI인가:** "화장품 인허가는 여전히 사람의 경험·노하우에 의존하는 비효율이 많다는 걸 JD를 보고 공감했다. 나도 핀테크에서 레거시 수동 업무를 자동화하며 95% 이상 공수를 줄인 경험이 있어, 같은 방향의 문제를 IT로 해결하는 일에 매력을 느꼈다."
- **스타트업 문화 적합성:** "큐뱅에서 결제 시스템을 0에서 구축했고, 거래량이 100배 성장하는 과정에서 지속적으로 아키텍처를 개선해왔다. CDRI의 '시행착오를 통한 혁신' 문화와 맞다고 생각한다."
- **AI 협업 경험:** "단순히 AI API를 붙이는 게 아니라, 운영팀과 함께 AI 활용 구조를 설계하고 비개발 직군이 AI와 협업하는 환경을 만들어본 경험이 있다. CDRI의 AI 서비스 고도화 과정에서 같은 역할을 하고 싶다."
- **성장 방향:** "복잡한 도메인을 IT로 단순화하는 것에 관심이 있고, 화장품 산업 TIC 데이터를 구조화하는 과정에서 백엔드 설계 역량을 더 키우고 싶다."
