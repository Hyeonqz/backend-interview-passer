# 기업 비즈니스 조사 Skill

서류 합격 기업의 비즈니스를 과거부터 현재까지 조사하고, 사용자가 어떤 방식으로 회사 성장에 기여할 수 있는지 분석한다.
결과물은 기술면접과 인성면접 모두에서 실제로 활용할 수 있는 수준이어야 한다.

## 소스 파일 (반드시 먼저 읽을 것)

| 파일 | 용도 |
|------|------|
| `.claude/JD.md` | 지원 JD — 회사명, 직무, 요구 기술, 팀 소개 등 |
| `.claude/resume/resume.html` | 이력서 — 보유 기술·프로젝트·성과 |
| `.claude/resume/career.html` | 경력기술서 — 프로젝트별 상세 경험 |

## 데이터 관리 구조

```
기업조사/
└── {회사명}/
    └── research.md     # 조사 결과 리포트 (실행할 때마다 최신화)
```

## Workflow

### Step 1. JD 분석

`.claude/JD.md`를 읽어 다음 정보를 추출한다.

- 회사명 / 서비스명 / 지원 직무·팀
- JD에서 강조하는 기술 키워드 (예: Kafka, MSA, 대용량 트래픽)
- 회사가 직접 언급한 도전 과제 또는 성장 방향
- 팀 문화·협업 방식 단서

### Step 2. 사용자 역량 파악

이력서와 경력기술서를 읽어 핵심 기술, 프로젝트 경험, 정량적 성과를 정리한다.
Step 4의 기여 분석에서 JD·회사 방향과 교차 분석한다.

### Step 3. 회사 비즈니스 조사 (WebSearch 필수)

WebSearch로 확인된 사실만 수집한다. 확인되지 않은 내용은 "(미확인)"으로 표기하고, 내용을 지어내지 않는다.

**검색 순서와 키워드**

| 조사 항목 | 검색 키워드 |
|-----------|------------|
| 창업 배경·설립 | `{회사명} 설립 연도 창업 배경 창업자` |
| 초기 비즈니스 모델 | `{회사명} 초기 서비스 사업모델` |
| 성장·서비스 확장 이력 | `{회사명} 서비스 출시 연혁 히스토리` |
| 현재 주력 사업 | `{회사명} 주요 서비스 현재 사업` |
| 최근 동향·전략 | `{회사명} 2024 2025 뉴스 신사업 전략` |
| 기술 스택·과제 | `{회사명} 테크 블로그 기술 스택 아키텍처` |
| 투자·재무 개요 | `{회사명} 투자 유치 시리즈 매출 IPO 기업가치` |
| 시장 위치·경쟁 | `{회사명} 경쟁사 시장 점유율 업계 위치` |

재무 정보는 **개요 수준(투자 라운드, 기업가치 규모, 흑자/적자 여부 정도)** 만 기록한다. 정확한 수치가 확인되지 않으면 추측하지 않는다.

### Step 4. 기여 가능성 분석

사용자 역량(Step 2)과 회사 비즈니스(Step 3)를 교차 분석하여 "내가 이 회사의 성장에 어떻게 기여할 수 있는가"를 구체화한다.

분석 관점:
- 회사가 현재 풀고 있는 기술 과제 중 내가 직접 경험한 영역
- JD에서 요구하는 기술·역량 중 내 이력서와 겹치는 지점
- 회사의 사업 성장 방향에서 내가 맡을 수 있는 역할
- 내 경험 중 이 회사 맥락에서 특히 강조할 수 있는 것

### Step 5. 면접 활용 포인트 도출

기술면접과 인성면접 각각에서 쓸 수 있는 포인트를 정리한다.

- **기술면접**: 회사 기술 환경에 대한 이해, 내 경험과의 연결고리
- **인성면접**: 회사 비즈니스를 선택한 이유, 입사 후 기여 계획, 성장 방향

## 품질 기준

- **거짓 정보 금지**: 검색으로 확인되지 않은 내용은 작성하지 않는다. 불확실하면 "(미확인)" 표기.
- **재무는 rough하게**: 정확한 수치 집착 없이 규모감과 방향성을 전달하는 수준으로 작성.
- **사용자 경험 왜곡 금지**: 기여 분석 시 이력서에 없는 경험을 부풀리지 않는다.
- **면접 실용성 우선**: 외우기 쉽고 면접에서 바로 꺼낼 수 있는 형태로 작성한다.
- **최신 정보 우선**: 2024~2025년 기준 정보를 수집한다.
