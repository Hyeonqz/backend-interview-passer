# 기업 비즈니스 조사 Skill

서류 합격 기업을 대상으로 JD를 분석하고, 회사의 비즈니스를 과거부터 현재까지 조사하여 면접 준비에 활용할 수 있는 리포트를 작성한다.

## 소스 파일 (반드시 먼저 읽을 것)

| 파일 | 용도 |
|------|------|
| `.claude/JD.md` | 지원 JD — 회사명, 직무, 요구 기술, 팀 소개 등 |
| `.claude/resume/resume.html` | 이력서 — 보유 기술·프로젝트·성과 |
| `.claude/resume/career.html` | 경력기술서 — 프로젝트별 상세 경험 |

## 데이터 관리 구조

```
기업조사/
└── {회사명}/
    └── research.md     # 조사 결과 리포트 (실행할 때마다 최신화)
```

디렉토리명 예시: `기업조사/카카오뱅크/`, `기업조사/토스/`

## Workflow

### Step 1. JD 분석

`.claude/JD.md`를 읽어 다음 정보를 추출한다.

- **회사명 / 서비스명**
- **지원 직무 및 팀**
- **JD에서 강조하는 기술 키워드** (예: Kafka, MSA, Spring, 대용량 트래픽 등)
- **회사가 언급한 도전 과제 / 현재 문제** (예: "빠른 성장에 따른 시스템 고도화")
- **팀 문화 / 협업 방식 단서**

### Step 2. 사용자 역량 파악

이력서와 경력기술서를 읽어 사용자의 핵심 기술, 경험, 성과를 파악한다.
이후 Step 4의 기여 가능성 분석에서 JD와 교차 분석한다.

### Step 3. 회사 비즈니스 조사 (WebSearch 필수)

WebSearch 도구를 사용해 검증된 사실만 수집한다. **추측이나 추론으로 내용을 채우지 않는다.**
확인이 안 되는 내용은 "(미확인)" 또는 "(출처 없음)"으로 표기한다.

조사 순서와 검색 키워드:

| 조사 항목 | 검색 키워드 예시 |
|-----------|-----------------|
| 창업 배경 / 설립 연도 | `{회사명} 설립 연도 창업 배경` |
| 초기 비즈니스 모델 | `{회사명} 초기 서비스 사업모델` |
| 사업 확장 이력 | `{회사명} 서비스 출시 연혁 히스토리` |
| 현재 주력 사업 | `{회사명} 주요 서비스 현재 사업` |
| 최근 동향 / 이슈 | `{회사명} 2024 2025 뉴스 신사업` |
| 기술 스택 / 테크 블로그 | `{회사명} 테크 블로그 기술 스택` |
| 투자 / 재무 현황 | `{회사명} 투자 유치 매출 IPO` |
| 경쟁사 / 시장 위치 | `{회사명} 경쟁사 시장점유율` |

### Step 4. 기여 가능성 분석

Step 2에서 파악한 사용자 역량과 Step 3에서 조사한 회사 비즈니스를 교차 분석한다.

분석 관점:
- JD의 기술 요구사항 중 사용자가 직접 경험한 영역
- 회사가 현재 풀고 있는 기술 과제와 사용자의 경험이 맞닿는 지점
- 회사의 성장 방향에서 사용자가 기여할 수 있는 역할
- 사용자 경험 중 이 회사에서 특히 가치 있을 것들

## 품질 기준

- **거짓 정보 금지**: 검색으로 확인되지 않은 내용은 작성하지 않는다. 불확실한 내용은 "(미확인)" 표기.
- **출처 명시**: 수치·날짜·사실 주장에는 근거 출처를 명시한다.
- **사용자 경험 왜곡 금지**: 기여 분석 시 이력서에 없는 경험을 부풀리지 않는다.
- **최신 정보 우선**: 가능하면 2024~2025년 기준 정보를 수집한다.
- **모르면 모른다고**: 정보가 없으면 "공개된 정보 없음"으로 표기하고 넘어간다.
