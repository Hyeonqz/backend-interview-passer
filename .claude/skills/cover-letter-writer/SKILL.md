# 자기소개서 작성 Skill

사용자의 실제 경험을 기반으로 기업별 자기소개서를 함께 작성하고 관리하는 Skill이다.

## 소스 파일 (반드시 먼저 읽을 것)

자소서 작성 전 아래 두 파일을 반드시 읽어 사용자의 실제 경험을 파악한다.

| 파일 | 용도 |
|------|------|
| `.claude/resume.md` | 이력서 — 프로젝트·기술스택·성과 요약 |
| `.claude/resume/career.html` | 경력기술서 — 프로젝트별 상세 배경/문제/해결/성과 |

## 데이터 관리 구조

```
자기소개서/
├── {회사명}_{직무}/
│   ├── questions.md     # 자소서 항목 원문 (첫 실행 시 생성)
│   ├── v1.md           # 초안
│   ├── v2.md           # 1차 수정본
│   └── final.md        # 최종 확정본 (사용자가 "최종"이라고 할 때)
└── ...
```

디렉토리명 예시: `카카오뱅크_금융회계_백엔드`, `토스_서버개발`, `네이버_백엔드`

## Workflow

### Step 1. 정보 수집

사용자로부터 아래 정보를 파악한다. 이미 대화에서 제공된 정보는 다시 묻지 않는다.

- **회사명** (예: 카카오뱅크)
- **지원 직무** (예: 금융회계 시스템 백엔드 개발자)
- **자소서 항목** — 항목 번호, 원문, 글자수 제한
- **JD 키워드** (있으면) — 회사가 강조하는 가치·기술·문화

### Step 2. 초안 작성

이력서와 경력기술서를 읽은 뒤, 아래 원칙으로 초안을 작성한다.

**작성 원칙**
- 실제 경험에서 나온 구체적 수치와 사례를 반드시 포함한다 (예: "CS 95% 감소", "응답시간 85% 개선")
- 지원하는 기업의 특성·가치와 사용자의 경험을 명확하게 연결한다
- 글자수 제한의 85~97% 수준으로 작성한다 (여유를 두되 충분히 활용)
- 스펙 나열이 아닌 '문제→해결→성과' 스토리 구조로 서술한다
- 금융·핀테크 직무라면 정합성, 안정성, 컴플라이언스 관련 경험을 앞세운다

**자소서 항목 유형별 접근**

| 항목 유형 | 핵심 구조 |
|-----------|-----------|
| 지원 동기 / 입사 이유 | 내 경험에서 나온 신념 → 이 회사에서만 실현 가능한 이유 → 구체적 목표 |
| 직무 적합성 / 역량 | 3개 역량을 수치 근거로 제시 → 해당 직무와의 직접 연결 |
| 성장 과정 / 가치관 | 결정적 경험 → 깨달음 → 지금의 일하는 방식으로 연결 |
| 협업 / 팀워크 | 구체적 상황 → 내 역할 → 결과 → 배운 것 |

### Step 3. 초안 저장

작성 후 반드시 파일로 저장한다.

```
자기소개서/{회사명}_{직무}/v1.md
```

파일 내부 구조:

```markdown
# {회사명} 자기소개서 — v{N}
작성일: {날짜}  
글자수: {각 항목 글자수} / {제한}

---

## Q1. {항목 원문}

{답변 내용}

---

## Q2. {항목 원문}

{답변 내용}
```

### Step 4. 수정 반복

초안을 보여준 후 사용자의 피드백을 받는다. 수정 시:
- 수정 이유를 한 줄로 설명한 뒤 수정본을 제시한다
- 수정본은 v2, v3... 파일로 저장한다
- 이전 버전은 삭제하지 않는다

사용자가 "최종", "확정", "완성"이라고 하면 `final.md`로 복사하여 저장한다.

## 글자수 계산 방법

한국어 자소서에서 '1000자'는 공백 포함 1000자를 의미하는 경우가 대부분이다.
초안 작성 시 항목 아래에 `[글자수: XXX / 1000]` 형태로 표기한다.

## 주의사항

- 실제 이력서에 없는 경험은 절대 추가하지 않는다
- 추상적·진부한 표현("열정", "최선을 다하겠습니다")은 구체적 경험으로 대체한다
- 동일 경험이라도 기업별로 강조 포인트를 달리 적용한다
- 금융권 지원 시 안정성·정확성·컴플라이언스 관련 경험을 우선 배치한다
