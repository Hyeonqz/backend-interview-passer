기술적 개선 사항

MSA 분산 환경 실시간 Push 아키텍처 설계 — Kafka 기반 채널 소유권 문제 해결

배경 MSA 환경에서 결제 완료 후 POS 단말에 결과를 Push해야 하나, TCP 소켓 특성상 Netty Channel은 연결을 맺은 서버에만 존재— API 서버에서 직접 Push 불가, 분산 인스턴스 간 채널 소유권 불일치 문제 발생
해결 API 서버는 Kafka produce만 담당, TCP 서버 전체가 인스턴스별 고유 groupId로 consume 후 Redis 기반 소유 서버 판별(isOwnedHost)로 Channel 보유 서버만 Push 실행
  • 전략 패턴(TcpPushHandler) 적용으로 파트너사 추가 시 기존 코드 수정 없이 확장 가능한 구조 설계.
  • Channel 미존재(소켓 단절) 케이스는 망취소 자동 처리로 데이터 정합성 확보
결과 API 서버 ↔ TCP 서버 완전 디커플링 중복 Push 0건   장애 격리 구조 확보   [상세보기]

배경 신규 가맹점 접수 시 한글명을 영문으로 직역(예: 코레일유통_화장품점 → KORAILUTONG_HWAJANGPUMJUM)으로 자동 생성하여 가독성 저하 → 운영 담당자가 B/O에서 월 약 100건을 수동 수정하며, 담당자 부재 시 누락·지연 발생
