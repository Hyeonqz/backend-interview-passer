## Q1)
> 실제 운영 중인 시스템에서 모듈을 분리할 때 배포 순서와 롤백 전략이 중요해요. <br>
> 예를 들어 module-common에서 특정 유틸 클래스를 module-merchant로 이동할 때, <br>
> 기존 코드를 참조하는 다른 모듈들이 있는 상황에서 무중단으로 분리하려면 어떤 순서로 진행하셨나요?

## 모범 답변
```text
상황:
module-common의 MerchantUtils를 module-merchant로 이동
기존에 module-payment, module-settlement가 이를 참조 중

잘못된 방법:
1. module-common에서 MerchantUtils 삭제
2. module-merchant로 이동
3. 전체 배포
→ 배포 순서 어긋나면 참조 오류 발생


올바른 무중단 분리 순서:

Step 1. module-merchant에 MerchantUtils 복사 (중복 허용)
        module-common에도 기존 코드 유지

Step 2. module-payment, module-settlement가
        module-merchant의 MerchantUtils를 참조하도록 변경
        → 각 모듈 순차 배포

Step 3. 모든 모듈이 module-merchant 참조로 전환 확인 후
        module-common에서 MerchantUtils 삭제
        → 최종 배포

핵심 원칙:
삭제보다 추가가 먼저
참조 전환 확인 후 삭제
한 번에 하나씩 이동


롤백 전략:
각 Step을 독립적인 PR/배포로 분리
Step 2 롤백 시 module-common 참조로 즉시 되돌릴 수 있게
feature flag 또는 import 경로만 변경하는 구조 유지
```
